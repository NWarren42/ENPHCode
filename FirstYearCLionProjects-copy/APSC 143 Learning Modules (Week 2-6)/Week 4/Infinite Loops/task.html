Infinite Loops

One of the more common mistakes when programming repetition structures is to accidentally create an infinite loop, where the exit condition is not reachable. Naturally, this causes your code to run indefinitely until stopped manually.

The example code, which simulates exponential decay, displays such an example of an infinite loop. On each iteration, the population variable pop is updated by multiplying itself by 0.99^i, where i is the loop counter variable.

To fix this problem, think about the population variable as representing the population of a city or town - the loop should exit when there is no one left in the population. The hint should help if you're still confused.

If the population is less than 1, there is no one left so we should stop looping.

Note that there is no Peek Solution for this task. Whenever you write a new loop, make sure an exit condition is always reachable!
