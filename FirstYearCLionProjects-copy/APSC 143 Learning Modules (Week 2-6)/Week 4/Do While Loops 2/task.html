Do-While Loops

Comparison to While Loops

This task will investigate an example of an edge case which sets these two loops apart: when the exit condition for both loops trigger immediately.

You will have to write both a while loop and a do-while loop for this task, which will both have identical operations. The provided code has an integer x and 2 counting variables: whileCount for the while loop, and doCount for the do-while loop. Both loops will have the following properties:

    Inside the loop, increment the counting variable. Then, update the value of x to be half its current value.
    The loop should run if x is an even value.

If you need help putting the loops together, see the hints below. When you run the code, the following output should occur:

While loop ran 0 times.

Do-while loop ran 2 times.

Does this output make sense? You are encouraged to experiment and see why the difference between the two loops is so significant here. Can you think of another example where a do-while loop would be preferable?

while (x is even) { increment whileCount; x /= 2; }
do { increment doCount; x /= 2; } while (x is even);
